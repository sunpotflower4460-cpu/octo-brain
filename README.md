# 🐙 OctoBrain

**8つの軽量LLMノードが並列で考え、統合脳がまとめる。「安いのに深い」思考エンジン。**

タコは、ニューロンの約3分の2が腕にある。腕が局所で判断し、中央脳は統合に徹する。
OctoBrainはこの構造をそのまま実装する — 8本の腕(超軽量モデル)+ 1つの頭(統合モデル)。

## 設計の3原則

このプロジェクトの品質とコストは、次の3つで決まる。実装中に迷ったら必ずここに戻ること。

### 1. 多様性は「人格」ではなく「作業」で作る
8ノードは「論理的な人格」「感情的な人格」ではない。それは同じモデルに8つの衣装を着せるだけで、出力の相関が高い。
各ノードには**狭くて検証可能な動詞**を割り当てる(前提を抽出する / 反例を1つ作る / 欠けている情報を列挙する…)。
軽量モデルは「自由に賢く」は苦手だが「狭く正確に」は強い。
UI上は「論理」「批判」などの視点の世界観で見せ、内部プロンプトは動詞ベース — **表と裏を分離する**。

### 2. 天井は統合脳が決める
Mixture-of-Agents系の知見では、最終品質を支配するのは統合役の質。予算を使う唯一の場所は統合。
統合プロンプトは手続き化する(矛盾検出 → 確信度で重み付け → 弱い報告の棄却 → 合成)。
質の低いノード出力は**捨てる勇気**を持つ(クォーラム設計)。最後に軽量モデルで検証を1回。

### 3. 原価の主犯は出力ではなく「入力の複製」
会話履歴全文×8ノードが一番高くつく。ローリング要約1本を全ノードで共有。
システムプロンプトは固定してプロンプトキャッシュに乗せる。
max_tokensは「お願い」ではなくAPIパラメータで物理的に縛る。

## アーキテクチャ

```
iPhone / Web アプリ (React + Vite + Capacitor)
        ↓ SSE
Cloudflare Workers + Hono  ← APIキーはここだけ (wrangler secret)
        ↓
   ① Router      … 質問の重さを分類 (simple/normal/complex) → 起動ノード数を決定
   ② Nodes ×2〜8 … 動詞ノードを並列実行 (allSettled + timeout + クォーラム)
   ③ Synthesizer … 統合脳。手続き化プロンプトで合成 (ストリーミング出力)
   ④ Verifier    … 矛盾・過剰断定・安全の軽量チェック (表面修正のみ)
        ↓
   KV: クォータ管理 + 原価ログ (全モデル呼び出しが必ず通る)
```

## モデル選定について (重要)

このリポジトリのドキュメントには**具体的なモデルIDを書いていない**。
LLMの価格とラインナップは変動が激しく、設計書に焼き付けると腐るため。

モデルIDは `workers/src/config/models.ts` に一元化する (P0で作成)。
実装時に最新の価格を確認して埋めること。選定の目安:

| 役割 | 求める性質 | 価格帯の目安 |
|---|---|---|
| Router | 3択分類のみ。最安でよい | 超軽量 |
| Nodes ×8 | JSON出力が安定する軽量モデル。2〜3社に分散すると多様性が上がる | 軽量〜超軽量 |
| Synthesizer | 統合品質がアプリの天井。中位〜やや上位 | 中位 |
| Verifier | 表面チェックのみ | 軽量 |
