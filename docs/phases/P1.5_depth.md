# P1.5 — 深化アーキテクチャ: 八芒星のタコ

## 目的

OctoBrainを「広さの機械」から「広さ+深さの機械」へ拡張する。
8レンズを八芒星配置(2つの正方形+4つの緊張軸)に再設計し、統合脳を「まとめ」から
「掘り」へ改訂し、タコの腕間結合を模した深化エンドポイントを追加する。

前提: P0〜P1完了(mainに取り込み済み)。設計の全体像と背景は **`docs/01_depth_design.md`** を必読。
このファイルはその実装指示。設計思想で迷ったら 01_depth_design.md が正。

## スコープ

P1で作った `workers/` のノード定義・スキーマ・統合ロジックの**拡張**。
新規エンドポイント `/api/deepen` の追加。フロントエンドは触らない(P2拡張は別途)。
これは新規構築ではなく既存P1コードの改訂を含むため、既存テストの更新も範囲に入る。

## 絶対ルール(再掲・厳守)

- モデル名は `models.ts` のみ。内部verbとUI名は `nodes.ts`(と表側)で分離
- 全LLM呼び出しは `callModel`/`callModelStream` 経由(deepenの再考コールも含む)。原価ログを通す
- 追加するプロンプトはすべて**固定文**にする(キャッシュ維持。変動情報はuserメッセージ側へ)

## 実装内容

### 1. 8レンズの再設計 `src/config/nodes.ts`

`docs/01_depth_design.md` §4.1 の表のとおり、8レンズを差し替える。各レンズに以下を持たせる:

```ts
interface Lens {
  id: string;          // "reason" | "emotion" | "risk" | "empathy" | "future" | "truth" | "step" | "values"
  verb: string;        // 内部プロンプト(動詞・固定・全ドメイン共通)
  uiName: string;      // 表示名(論理/心/盾/友/望遠/鏡/一歩/核)
  emoji: string;
  square: "see" | "feel";   // 見る四角 / 感じる四角
  axis: string;             // 対角ペアのID(下記)
}
```

8レンズと対角軸(§3.2):
- **時の軸**: `reason`(合理・see) ⇔ `future`(未来・see)
- **心の軸**: `emotion`(感情・feel) ⇔ `truth`(真実・feel)
- **動の軸**: `risk`(リスク・see) ⇔ `step`(次の一歩・see)
- **魂の軸**: `empathy`(共感・feel) ⇔ `values`(価値観・feel)

内部verbは §4.1 の「内部動詞」列をそのまま使う。全ドメイン共通・完全固定。

### 2. ノード出力スキーマ移行 points → opinions

**これはP1からの破壊的変更。** 移行元(現状): `{points:[...], confidence, flag}`。
移行先(§4.1):

```json
{
  "opinions": [ { "claim": "60字以内", "weight": 0.9, "why": "60字以内" } ],
  "flag": null
}
```

- opinionsは1〜3個。各claim/whyは60字以内(超過は切り詰め)。weightは0〜1にクランプ
- 空の場合・flag付きの扱いは従来踏襲(insufficient_input / off_topic)
- 共通systemプロンプト(NODE_SYSTEM)とNODE_OUTPUT_FORMATを新スキーマに合わせて書き換え
- **パーサ更新** `runNodes.ts`: opinions配列のパース・各要素のクランプ・不正要素の除去。
  JSON抽出フォールバック(生パース→`{...}`抽出1回→parse_error)は維持
- 軽量モデルがネストで崩れやすい点に配慮: opinionsはフラット・最大3・キー名固定を厳守させる

### 3. 統合脳の改訂 `src/lib/synthesize.ts`

§5 の「掘る版」systemプロンプトに全面改訂(固定文)。骨子は§5の手順1〜8をそのまま実装:
- 入力から最も重い一文を引用して開始
- 4軸それぞれで対角2レポートの張り合いを検出
- 最も張り詰めた軸を特定し、本人が迫られている本当の選択を言語化
- 一般論の削除・weight<0.4は参考・flag付き除外(従来ルール継続)
- 末尾に**まだ言葉にされていない問いを1つ**
- 機械可読行 `---TENSION--- {"axis":"心の軸","reason":"..."}` を `---SUMMARY---` の**前**に出力

buildReportsのuser側入力構築は opinions 形式に対応させる。

### 4. TENSION抽出

`---SUMMARY---` と同じCutterパターンでTENSIONも抽出する。
- ストリーム版(P2で実装済みのSummaryStreamCutter)に **TENSIONも切り出す**拡張、
  または同方式の別Cutter。マーカー順は `本文 → ---TENSION--- → ---SUMMARY---`
- 一括JSON版(`splitAnswerAndSummary`相当)もTENSION対応に
- **TENSION欠落時は非致命**: `meta.tension = null` とし、後続の深化ボタンは非表示になる(フロント判断)。
  パース失敗で全体を落とさない

### 5. Router拡張 `src/lib/router.ts`

§4.3・§6 に沿って:
- `plan` パラメータ追加: `"light"`(無料・2軸4腕) | `"deep"`(有料・4軸8腕)
- lightのとき、ドメインに応じた2軸を選択して該当4腕のみ起動
  - ドメイン判定は既存のRouter分類を拡張(simple/normal/complexとは別軸のドメインタグ:
    恋愛/仕事/金銭/家族/自分/一般)。判定プロンプトは固定文
  - 軸マッピング(§4.3): 恋愛→心+魂、仕事→時+動、金銭→時+動、家族→心+魂、自分→心+魂、一般→心+動
- deepは全8腕(従来complex相当)
- `plan` 明示がなければ light をデフォルト(安全側)

### 6. 深化エンドポイント `src/lib/deepen.ts` + `POST /api/deepen`

§6「深化ボタン」= タコの腕間結合の実装。

リクエスト: `{ input, summary, tension: {axis}, priorAnswer, clientId }`
処理:
1. tension.axis から対角2レンズを特定(nodes.tsのaxisフィールドで引く)
2. その2腕に**互いの元レポートを渡して再考させる**(固定テンプレ+相手レポートはuser側):
   ```
   あなたは{自分}の腕。対角の{相手}の腕はこう述べた: {相手のopinions}
   これを踏まえ、(a)譲れない点 (b)相手を読んで見方が変わった点 を出せ
   ```
   2コール・Flash・max_tokens厳守
3. 中央脳(Pro)が2つの再考を受けて織り直す(専用の固定プロンプト)
4. レスポンス: `{ answer, meta:{ axis, calls, totalCost, ms } }`

- 元の1パス目は**触らない**(独立並列・キャッシュ維持)。深化は完全に別リクエスト
- costlogを通す。KVに原価記録

### 7. API契約更新

`POST /api/analyze`(一括)と `/api/analyze/stream`(SSE)のレスポンス `meta` に追加:
- `meta.plan`: "light" | "deep"
- `meta.domain`: ドメインタグ
- `meta.tension`: `{axis, reason}` | null
- `nodes[].opinions`(points から変更)

`/api/deepen` は§6のとおり新規。

### 8. テスト更新・追加(Vitest・外部APIモック)

- opinionsパース: 正常/3超過切り詰め/weightクランプ/不正要素除去/parse_error維持
- TENSION抽出: あり/なし(欠落時null)/ストリーム分割またぎ(1文字ずつでも漏れない — 既存Cutterテストに倣う)
- Router: plan=light時に2軸4腕のみ起動/ドメイン→軸マッピング/plan省略時light
- deepen: axisから正しい対角2腕を選択/2腕再考→再統合の流れ/costlog計上/tension欠落時のガード
- 既存テスト(analyze/runNodes/synthesize/costlog)をopinions移行に追随して更新
- 全緑必須

## 受け入れ条件

- [ ] `.dev.vars` にキーを入れた状態で、`plan:"deep"` の `/api/analyze` が opinions 形式のノード出力と、
      掘る統合(引用開始・緊張の言語化・最後の問い)を含む回答を返す(手順を workers/README.md に追記)
- [ ] レスポンス `meta` に plan/domain/tension が入る。deep で8腕、light で4腕が起動する
- [ ] `---TENSION---` がストリーム・一括の両方で正しく切り出され、ユーザー向け本文に漏れない
- [ ] TENSION欠落時も全体が落ちず `meta.tension:null` になる
- [ ] `/api/deepen` が最緊張軸の2腕を再考させ、中央脳が織り直した回答を返す。KVに原価記録
- [ ] 深化コールを含む全LLM呼び出しが callModel 経由(原価ログを通る)
- [ ] Vitest 全緑・tsc 通過
- [ ] モデル名ハードコードなし・シークレット実値なし

## やらないこと

フロントのTENSION表示・深化ボタン・対角の腕が光るアニメーション(P2拡張として別途)。
ドメイン別レンズ変種(束の言い換え。まずは共通8レンズで動かす)。認証/IAP。
