# P1.6 — 共鳴: 掛け算の腕(バックエンド)

## 目的

深化(TENSION=対立から掘る)と対になる**共鳴(RESONANCE=結合から生む)**を実装する。
遠く離れた2つのopinionが響き合うとき、それを検出し、掛け合わせて第三の選択肢を生む。
創造性研究で bisociation(二重連想)と呼ばれる機構の実装。

前提: P1.5完了(mainに取り込み済み)。設計思想の文脈は `docs/01_depth_design.md`(八芒星・
TENSIONの対概念として読む)。幾何学的対応: {8/2}の対角線=TENSION、**{8/3}の一筆書き=RESONANCE**
(離れた点同士を結ぶ線)。

## スコープ

`workers/` のみ。統合脳プロンプトへの追記・マーカー抽出の拡張・新エンドポイント1本。
フロントは P2.6 で別途。

## 絶対ルール(再掲)

- 全LLM呼び出しは callModel 経由・costlog を通す
- 追加プロンプトはすべて固定文(キャッシュ維持。変動情報はuser側)
- モデル名は models.ts のみ(**新roleは追加しない**。共鳴の織り直しは synth ロールを流用)

---

## 実装内容

### 1. 統合脳プロンプトに RESONANCE 検出を追記 `src/lib/synthesize.ts`

掘る手順(§5)に1項目追加(固定文):

```
8. 軸をまたいで、遠いのに響き合う opinion の組がひとつだけあれば、機械可読行
   ---RESONANCE--- {"a":{"lens":"...","claim":"..."},"b":{"lens":"...","claim":"..."},"root":"共通の根を一文で"}
   を出力する。基準: (a)異なる軸に属する (b)共通の根が一文で言える
   (c)組み合わせると新しい選択肢が生まれる。3つすべて満たすときだけ。
   無理に作らない。該当が無ければこの行を出さない。
```

- **マーカー順(厳守)**: `本文 → ---RESONANCE---(任意) → ---TENSION--- → ---SUMMARY---`
- lens値は8つのNodeIdのいずれか。claimは元レポートのclaimを引用(60字以内)

### 2. マーカー抽出の拡張

- 一括版 `splitAnswerTensionSummary` → RESONANCE も切り出す(欠落時 null・非致命)
- ストリーム版 Cutter → RESONANCE 対応。**1文字ずつの分割でも本文に漏れない**
  (既存 TENSION/SUMMARY テストと同型のケースを必ず追加)
- パース失敗・lens不正(NodeId外)は `meta.resonance = null` に落とす(全体を止めない)

### 3. API契約: analyze レスポンスに追加

- `meta.resonance`: `{a:{lens,claim}, b:{lens,claim}, root}` | `null`
- SSE `done` イベントにも同様に含める

### 4. 新エンドポイント `POST /api/resonate` + `src/lib/resonate.ts`

リクエスト:
```json
{ "input": "...", "summary": "...", 
  "resonance": { "a": {"lens":"risk","claim":"..."}, "b": {"lens":"values","claim":"..."} },
  "priorAnswer": "...", "clientId": "..." }
```

- **AIが提案したペアでも、ユーザーが自分で選んだ2つのopinionでも、同じ形で受ける**
  (掛け算の主導権はユーザー側にもある — これが道具としての誠実さ)
- バリデーション: lens は NodeId として実在すること(不正は400)、claim は各120字以内、
  a と b が同一 lens なら 400
- 処理: **1コール**(synth ロール = Pro)。固定システムプロンプト:

```
あなたはOctoBrainの中央脳。ユーザーの状況に対し、2本の腕が出した一見遠い2つの意見を
掛け合わせる。手順:
1. 2つに共通する根をひとつ言い当てる
2. 組み合わせて生まれる第三の選択肢を具体的に描く(どちらか一方では出ない案であること)
3. それがこの人の状況で明日どう見えるか、最小の一歩に落とす
- こじつけない。根が繋がらない場合は正直に「この2つは独立している」と述べ、
  それぞれを別々に活かす形を短く示す
- 一般論禁止。この人の言葉に接地させる
```

- user側: `[会話要約(あれば)] + [今回の入力] + [priorAnswer(あれば)] + [aとbのlens/claim]`
- max_tokens: 800。costlog 必須
- レスポンス: `{ answer, meta: { pair:{a,b}, calls, totalCost, ms } }`

### 5. テスト(Vitest・モック)

- RESONANCE抽出: あり / なし(null) / 1文字ずつ分割またぎ / lens不正→null
- マーカー3連(RESONANCE+TENSION+SUMMARY)が正順で全部切れる / RESONANCE欠落でも他2つは切れる
- resonate: 正常フロー / 不正lens 400 / 同一lens 400 / claim超過 400 / costlog計上
- 既存テストの非破壊(全緑)

## 受け入れ条件
