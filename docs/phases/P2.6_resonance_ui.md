# P2.6 — 共鳴: 掛け算のUI(フロントエンド)

## 目的

P1.6の共鳴機構をUIに出す。AIが見つけた響き合いの提示、**ユーザー自身が2つの意見を
選んで掛け算する操作**、そして離れた2本の腕が絡み合うタコの見せ場。

前提: P1.6完了。深化UI(P2拡張)の実装パターンを踏襲する。

## スコープ

`web/` のみ。ただし OctopusCanvas への prop 追加は deepenArms と同型の**非破壊拡張**に限る。

## 絶対ルール(再掲)

- OctopusCanvas: 新prop `resonanceArms?: number[] | null` を追加してよいが、
  **null のとき既存コードパスと完全同一**であること(deepenArms と同じ流儀・ルール4)
- UI表示名は nodeDisplay.ts の管轄。内部lens IDを画面に出さない

---

## 実装内容

### 1. 共鳴カード(AI提案の表示)

- `meta.resonance` が非nullのとき、回答の下に共鳴カードを表示:
  - 2つのopinion(lensのUI名+絵文字つき)が**線で結ばれた**見た目
  - `root`(共通の根)を結び目のラベルとして表示
  - 「**掛け合わせる**」ボタン → `/api/resonate` へ(AI提案ペアをそのまま送る)
- 深化ボタン(TENSION)と並ぶ配置。両方あるときは縦に並べ、深化=⚡緊張、共鳴=✨響き合い、と視覚的に区別

### 2. ユーザー選択の掛け算

- 「8つの視点を見る」展開内の各opinionカードを**選択可能**にする(タップでチェック)
- **2つ選ぶと「この2つを掛け合わせる」ボタンが活性化** → 選んだ {lens, claim} ペアで `/api/resonate`
- 同一lens内の2opinionは選択不可(ボタン非活性+短い説明)。3つ目を選んだら最初の選択を解除
- 掛け算結果は元回答の下に「共鳴: 論理 × 核」のような見出しつきで追記表示

### 3. タコの見せ場: 腕の絡み合い

- resonate 実行中、選ばれた2本の腕(lens→腕indexの対応は深化と同じマッピングを流用)を
  `resonanceArms` として OctopusCanvas に渡す
- 演出の方向性: 深化(対角2本が強く脈打ち・他は暗転)との**差別化**。共鳴は
  2本の腕の波が**同期していく**(位相が揃う)・光が2本の間を往復するように交互に強まる、等。
  紫(深化)に対して**シアン〜白系**の発光。実装詳細は任せるが「対立」ではなく「呼応」に見えること
- resonanceArms が null のとき描画は従来と完全同一(テスト観点)

### 4. 状態管理・API

- `lib/api.ts` に resonate 呼び出し追加(一括JSON・ストリーム不要)
- 実行中は入力を無効化し、インジケータ「2本の腕が響き合っています…」
- エラー時は非致命表示(元回答は残す)

### 5. テスト

- resonance メタの有無によるカード表示分岐
- opinion選択ロジック: 2つで活性/同一lens不可/3つ目で入れ替え
- resonanceArms null時に OctopusCanvas へ渡る props が従来と同値
- APIエラー時に元回答が保持される

## 受け入れ条件

- [ ] AI提案の共鳴カードが表示され、「掛け合わせる」で第三の選択肢が追記される(モックSSE/HTTPで確認)
- [ ] ユーザーが任意の2 opinionを選んで掛け算できる。同一lens選択は防がれる
- [ ] resonate中、2本の腕が「呼応」する演出で光る。null時は従来描画と完全同一
- [ ] 深化と共鳴が同一回答に共存できる(両ボタン表示・順に実行可能)
- [ ] `npm run build` 通過・テスト全緑

## やらないこと

掛け算結果のさらなる深化(連鎖)。共鳴履歴の保存。モバイルネイティブ調整(P7)。
