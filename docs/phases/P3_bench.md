# P3 — 20問ベンチ: 「安いのに深い」を数字にする

## 目的

同一の20問に対して **OctoBrain** と **単発上位モデル** の回答を取得し、
品質・コスト・レイテンシを並べた比較表を作る。主張を数字に変えるフェーズ。

前提: P1完了(P2は不要。一括JSON版 `/api/analyze` を使う)。

## 実装内容

- 境界2問は**負けてよい問題**。数学と最新情報での限界が正直に出るか(誤魔化さず「わからない」と言えるか)を見る。製品の売り文句の裏付けになる

### 2. 実行スクリプト `bench/run.ts`

- Node.jsスクリプト(tsx実行)。ローカルの `workers` dev サーバーを叩く
- 各問について:
  - A: `POST /api/analyze` (mode: "complex" 固定 — ベンチは最大構成で測る)
  - B: 単発上位モデル(`callModel` を流用、role追加 or modelOverrideで指定。モデルIDは `bench/config.ts` で設定)
- 記録: 回答全文 / totalCost / ms / quorum を `bench/raw/{id}_{a|b}.json` に保存
- レート制限対策で直列実行+ウェイト可

### 3. 評価スクリプト `bench/evaluate.ts`

- 評価者: **AとBのどちらにも使っていない別系統のモデル**(バイアス回避。`bench/config.ts` で指定)
- ブラインド評価: 回答の提示順をランダム化し、どちらがOctoBrainか伏せる
- ルーブリック(各1〜5点): 深さ / 多角性 / 実用性 / 正確さ / 簡潔さ
- 各問で2回答を同時に見せて相対採点。境界問題(19,20)は「限界を正直に認めたか」も記録
- 出力: `bench/raw/eval_{id}.json`

### 4. 集計 `bench/report.ts` → `bench/results.md`

生成する表:

| 指標 | OctoBrain | 単発上位 |
|---|---|---|
| 品質スコア平均(カテゴリ別+総合) | | |
| 1回答あたり平均コスト(USD) | | |
| 平均レイテンシ(ms) | | |
| 勝敗数(問別の総合点比較) | | |

- 末尾に「コスト比 × 品質比」の一言サマリ(例: コスト1/4で品質92%)と、境界問題の挙動メモ
- raw ディレクトリは `.gitignore`、results.md はコミットする

## 受け入れ条件

- [ ] `npm run bench` 一発で 実行→評価→results.md 生成まで走る(途中再開可: 既存rawはスキップ)
- [ ] results.md にカテゴリ別品質・コスト・レイテンシの表が揃っている
- [ ] 評価がブラインド(ログで提示順ランダム化を確認できる)
- [ ] 全20問のrawが保存されている

## やらないこと

人手評価UI / 継続的ベンチ基盤 / モデル総当たり比較 — 必要になったら別フェーズ。
