# App Store スクリーンショット

実機ではなく、実アプリ(モックバックエンド)を **6.7インチ相当(1290×2796)** で撮影した本番同等の素材。
表示テキストはデモ内容だが、レイアウト・世界観・状態表現は実ビルドそのまま。

## 収録(`iphone-6.7/`, 各 1290×2796 PNG)

| # | ファイル | 内容 | 訴求 |
|---|---|---|---|
| 1 | `01-onboarding.png` | オンボーディング(8本の腕) | 世界観が一目で伝わる |
| 2 | `02-hero.png` | 空状態 Hero + 3ステップ + 例 | 「8つの視点が同時に考える」価値提案 |
| 3 | `03-thinking.png` | 処理中(思考の計器 + 4段階) | 思考過程が見える独自体験 |
| 4 | `04-answer.png` | 回答(Insight Card + 次に、どう考える?) | 読みやすい回答 + 深化/共鳴 |
| 5 | `05-viewpoints.png` | 8つの視点グリッド | 多角性の可視化(独自アーキテクチャの証拠) |

## App Store Connect にアップロードするサイズ

- **必須: 6.7インチ (1290×2796)** — 本フォルダ。iPhone 15/16 Pro Max 等
- 任意: 6.5インチ (1242×2688)。多くの場合 6.7 のみで審査可

### 別サイズが必要になったら(再生成コマンド)

`scratchpad/store_shots.mjs` の `VP`(論理サイズ)を変えて再実行:

- 6.5インチ → `{ width: 414, height: 896 }`, `deviceScaleFactor: 3` → 1242×2688

## 注意

- 文言は差し替え可(実データや、キャッチコピー入り装飾フレームを被せてもよい)
- 「最強」「No.1」等の誇大表現は避ける(審査リスク)。P3 の数字ベースの表現を使う
