## STEP 1: プレースホルダを埋める（`SET_ME` / `[要記入]`）

埋めるべき箇所の**完全な一覧**。ファイルを開いて値を入れるだけ。

### 1-1. バックエンドのモデル（キー確定後・STEP2と一緒に）
| ファイル | 箇所 | 入れる値 |
|---|---|---|
| `workers/src/config/models.ts` | `router/node/synth/verifier` の `baseURL`・`model` | 実プロバイダーのURL・モデルID |
| 同上 | 各 `pricePerMTokIn/Out` | 最新の実価格（USD/100万トークン） |
| `workers/bench/config.ts` | `BENCH_SINGLE`・`BENCH_EVAL` の `baseURL`・`model`・価格 | ベンチ用（A/B/評価）※任意 |

> モデル名・価格は**このファイルだけ**で管理（他所に書かない = 絶対ルール2）。

### 1-2. アプリの外部リンク・連絡先
| ファイル | 箇所 | 入れる値 |
|---|---|---|
| `web/src/config/appInfo.ts` | `PRIVACY_URL` / `TERMS_URL`（`"SET_ME"`） | STEP3 でホスティングした公開URL |
| 同上 | `SUPPORT_EMAIL` | 既定は本人 gmail。変えたい場合のみ |
| `web/.env.production` | `VITE_API_BASE`（`SET_ME_PRODUCTION_WORKERS_URL`） | 本番 Workers の URL |

### 1-3. 法務・ストア文言
| ファイル | 箇所 | 入れる値 |
|---|---|---|

## STEP 2: バックエンドを本番化（要 LLM キー）

詳細 → `docs/release/ios_build_runbook.md` の「1. 本番API」。

1. **モデル確定（P4）**: キーを `.dev.vars` に置き、`cd workers && npm run bench`（本番=MOCK無し）
   → `bench/results.md` を読み、`models.ts` の `SET_ME`（モデル/価格）を確定。
   - `docs/phases/P4_tuning.md`（残タスク）と `docs/pricing_notes.md`（料金）を更新
   - **胸テスト**: 本物の悩みを3つ投げ、刺さるか自分の胸で判定（最終検証者は人）
2. **KV 作成 → `wrangler.toml` に id 反映**
   ```bash
   cd workers
   wrangler kv namespace create OCTO_KV          # 出た id を wrangler.toml の id= に
   wrangler kv namespace create OCTO_KV --preview # preview_id= に
   ```
3. **シークレット投入（リポジトリに書かない）**
   ```bash
   wrangler secret put DEEPSEEK_API_KEY          # 使うプロバイダー分
   ```
4. **デプロイ** → 出た URL を STEP1-2 の `VITE_API_BASE` に
   ```bash
   npm run deploy
   ```
   - CORS は `capacitor://localhost` 等を許可済み。通常 `ALLOWED_ORIGIN` 追加は不要
5. 動作確認: `curl https://<your-workers-url>/api/health` → `{"ok":true,...}`

## STEP 4: iOS ビルド → 実機 → TestFlight（要 Mac/Xcode/実機）

詳細 → `docs/release/ios_build_runbook.md`。要点だけ:

```bash
cd web
npm ci
npm run build                 # .env.production が効く
npx cap add ios               # 初回のみ
npx cap sync ios
npx @capacitor/assets generate --ios \
  --iconBackgroundColor '#050711' --splashBackgroundColor '#050711'
npx cap open ios              # Xcode 起動
```
Xcode で:
- Signing: Team 選択 / Bundle ID = `com.octobrain.app`
- Display Name = OctoBrain / Version = 1.0.0 / Build = 1
- **実機で確認**: 下部入力の Safe Area、キーボード表示、fps・発熱（ROADMAP P7 完了目安）
- Product > Archive → Distribute → App Store Connect → Upload → TestFlight 内部テスト

## つまずきポイント（早見表）

| 症状 | 原因 | 対処 |
|---|---|---|
| アプリが白画面 | `VITE_API_BASE` が SET_ME / `cap sync` 忘れ / `dist` 未生成 | STEP1-2・STEP4 を確認 |
| API が 401/403 | `wrangler secret` 未設定 / CORS | STEP2-3・キー確認 |
| 429 が出る | 正常（クォータ/連打防止）。UIに平易メッセージ | 上限や間隔は wrangler.toml で調整 |
| 審査 4.2 リジェクト | 最小機能と見なされた | 審査ノートの独自性＋8視点スクショを添付し再提出 |
| 審査 1.2 リジェクト | 生成物の報告手段 | 「問題を報告」導線＋Verifier を説明し再提出 |
| 17+ への変更要求 | 生成AIは保守的判定 | 素直に従う |
