## 1. 1回答あたりの実コスト(原価)

OctoBrain の1回答は次のモデル呼び出しの合計:

```
コスト/回 = router + Σ(node × 起動腕数) + synth + verify
          = Σ_call ( inTok/1e6 × pricePerMTokIn + outTok/1e6 × pricePerMTokOut )
```

- 単価は `workers/src/config/models.ts`(唯一の正)。**ドキュメントに価格を書かない**(絶対ルール2)
- 実測は原価ログから: `npm run ops:report -- --days 7`(総コスト・件数・平均ms・内訳)
- プラン別の腕数(`docs/01_depth_design.md §6`):
  - **light**(無料想定): 2軸4腕 + Flash統合 → 概算 `<実測>` 円/回
  - **deep**(Pro想定): 4軸8腕 + Pro統合 → 概算 `<実測>` 円/回
- **キャッシュヒット率**: DeepSeek の `prompt_cache_hit_tokens` を実測し、見積り(0.4〜0.75円/回)を置き換える → `<実測>`

## 2. 深化・共鳴の追加原価(プレミアム根拠)

- 深化 = 4コール(対角2腕の再取得 + 再考2 + 中央織り直し)/ 共鳴 = 1コール
- `ops:report` の内訳(`kind: deepen / resonate`)で **使用率と追加コスト**を実測 → `<実測>`
- プレミアム(Pro)の象徴機能。サブスク下限の妥当性の根拠にする

## 3. 損益分岐と価格の考え方(枠組み)

- **無料枠**: 月 N 回 × light 原価 = 1ユーザーあたり月間原価上限。N は `FREE_MONTHLY_QUOTA`(現状100)
  - 無料の損益分岐: `月間インフラ許容額 / (light原価 × 想定アクティブ率)` → `<実測>`
- **Pro(月額)**: deep + 深化/共鳴 + 回数増。Apple 手数料(15〜30%)後で採算が出る下限を置く
  - 方針は「**サブスク下限は安く**」。ただし 480円台は手数料後ギリギリ → 深化/共鳴の実使用率(P4実測)を見て最終決定
- **リワード動画広告(非強制・補助)**: eCPM 実測まで収益は控えめに見積もる。青天井にせずサブスクと段差
