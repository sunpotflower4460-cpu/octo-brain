## 1. 「深い」の正体

浅い回答と深い回答を分解すると、差は情報量ではない。

| 浅い | 深い |
|---|---|
| 一般論(誰にでも当てはまる=誰にも刺さらない) | **この人の状況にしか成立しない文** |
| 質問に律儀に答える | **質問の背後の本当の問いを掘り当てる** |
| 視点が並ぶ(羅列) | 視点が**一つの理解に織り上がる** |
| 安全な距離から言う | **矛盾や目背けを、愛を持って指摘する** |
| 本人の言葉を言い換える | **本人の言葉の細部を拾い、まだ言語化されていないものを言い当てる** |

ここから出る設計上の結論:

**深さは視点の数からは生まれない。視点の"あいだ"から生まれる。**

8視点を並べるのは「広さ」。深さが立ち上がるのは、視点と視点が**張り合う場所**——合理はこう言うが感情はこう言っている、その緊張こそが「本人が迫られている本当の選択」を指し示す。だから深さの設計とは、**緊張を構造的に作り、それを掘る係を置くこと**。

---

## 2. 現状の正直な診断

いまのOctoBrainは**広さの機械**として正しく設計されている。8腕並列は見落としを防ぐ。単発AIに対する明確な優位。

しかし深さを担う係が**統合脳ひとり**で、その統合脳への指示は「矛盾を検出し、重み付けし、まとめよ」——つまり**「まとめ」であって「掘り」ではない**。материалを綺麗に整理する係はいるが、材料の奥に降りる係がいない。

さらに、8本の腕は互いを知らない。完全に独立並列。だから「視点のあいだ」が構造として存在せず、緊張は偶然にしか生まれない。

**結論: 深さは、腕の関係構造と、中央の掘る仕事の両方を設計しないと生まれない。**

## 3. 二つの師 — タコと八芒星

### 3.1 タコの神経設計から借りる3つ

本物のタコの神経系は、この設計の答えを先に持っている。

| タコの事実 | OctoBrainへの借用 |
|---|---|
| ニューロン約5億のうち約3分の2が8本の腕にある。腕は局所の神経節で独立に判断する | 思考の主力は8腕(軽量モデル)。各腕は狭い専門作業を自律実行 — **実装済み** |
| 腕から中央脳へは、全感覚データではなく**要約された情報**だけが上がる | JSONレポート(points/confidence)のみを統合脳へ — **実装済み。生物学的に正しかった** |
| 腕と腕のあいだには、**中央脳を経由しない直接の神経結合**があるとされる(腕同士が局所で調整し合う) | **未実装 — 今回の核心。** 腕は完全独立ではなく、対になる腕と関係を持つべき |
| 吸盤は「触る」と「味わう」を同時に行う(化学触覚受容体) | 観察(見る)と評価(感じる)を分離しない — レンズ配置に反映(後述) |
| 中央脳の垂直葉が学習と記憶を司る | 統合脳の「掘り」+ローリング要約+将来の記憶機構(P6) |

3つ目が今回の発見です。**腕間結合** — タコの腕は中央を介さずに互いに調整し合う。これをそのまま「レンズ同士の関係構造」として借りる。ただし8本総当たりの通信はコストが爆発するので、**対** に絞る。どの対か — それを決めるのが八芒星。

### 3.2 八芒星の幾何学 — 2つの正方形と4本の対角線

八芒星 {8/2} は、**2つの正方形が45度ずれて重なった形**。この構造がそのまま設計になる。

```
                 合理
                  ●
        価値観 ○     ○ 感情
ize
    次の一歩 ●           ● リスク
        
         真実 ○     ○ 共感
                  ●
                 未来

  ● = 見る四角(外の現実を見る): 合理・リスク・未来・次の一歩
  ○ = 感じる四角(内の心を感じる): 感情・共感・真実・価値観
```

**2つの正方形 = 広さの二重奏。**
「見る四角」は状況を捉える(事実・危険・先行き・行動)。「感じる四角」は心を捉える(感情・受容・直視・根)。円周上で交互に並ぶので、**隣は必ず異種** — 見ることと感じることが織物のように交差する。タコの吸盤が触りながら味わうように、このAIは観察しながら感じる。

**4本の対角線 = 深さの4軸。**
対角(180度)で向かい合うレンズは、**意図的に張り合う対**として設計する:

| 軸 | 対 | 緊張の中身 |
|---|---|---|
| **時の軸** | 合理 ⇔ 未来 | いま最適なことと、5年後に正しいことは、しばしば別 |
| **心の軸** | 感情 ⇔ 真実 | いま感じていることと、目を背けていることは、しばしば別 |
| **動の軸** | リスク ⇔ 次の一歩 | 止まる理由と、踏み出す理由は、同時に正しい |
| **魂の軸** | 共感 ⇔ 価値観 | そのままのあなたを認めることと、なりたいあなたから問うことは、両方が愛 |

この4軸が「視点のあいだ」を構造化する。統合脳は8個の独立意見ではなく、**4つの対話**を受け取ることになる。深さが偶然ではなく構造から生まれる。

## 4. 8レンズ最終設計

### 4.1 各レンズの定義

全レンズ共通の出力スキーマ(前回合意した opinions 形式):

```json
{
  "opinions": [
    { "claim": "60字以内の意見", "weight": 0.9, "why": "60字以内の理由" }
  ],
  "flag": null
}
```
opinionsは1〜3個。weightは0〜1の確信度。軽量モデルが崩れないようフラット構造・上限厳守。

| # | レンズ | 四角 | 内部動詞(systemに固定) | UI名(案) |
|---|---|---|---|---|
| 0 | 合理 | 見る | 感情を除き、事実と数字だけで状況を切り分ける | 論理 🧠 |
| 1 | 感情 | 感じる | 言葉の裏で本当に感じていることを探り当てる | 心 💧 |
| 2 | リスク | 見る | 見えていない危うさ、引き返せなくなる地点を見積もる | 盾 🛡️ |
| 3 | 共感 | 感じる | 本人の味方として、そのままの気持ちを受け止めて言葉にする | 友 🤝 |
| 4 | 未来 | 見る | 半年後・数年後、この選択がどう見えているかを描く | 望遠 🔭 |
| 5 | 真実 | 感じる | 本人が目を背けている可能性を、正直にひとつだけ指摘する | 鏡 🪞 |
| 6 | 次の一歩 | 見る | 明日できる最小の具体行動をひとつに絞る | 一歩 🎯 |
| 7 | 価値観 | 感じる | この人が本当に大切にしているものを、入力の言葉から掘り当てる | 核 💎 |

内部プロンプトは全ドメイン共通・完全固定(キャッシュ§7参照)。UI名・絵文字は表側の自由(nodeDisplay.tsの管轄)。

### 4.2 落選レンズの行き先(捨てない)

- **「本当の問い」→ 中央脳の専任にする。** 8本の報告を見渡した者にしか、本当の問いは見えない。タコで言えば垂直葉(学習と記憶の座)の仕事。統合手順の最終ステップに組み込む(§5)。腕にするより強い配置。
- **「相手の視点」「前例・相場」「倫理・誠実さ」→ ドメイン変種として温存。** 恋愛ドメインでは「共感」の動詞を「相手の立場から見る」に差し替える等、**束ごとの言い換え**で吸収(束数は増やしすぎない — キャッシュが温まる範囲で)。

### 4.3 ドメインとの関係

基本8レンズは全ドメイン共通。ドメイン(恋愛/仕事/金銭/家族/自分)が変えるのは:
1. **無料版でどの2軸を起動するか**(恋愛→心の軸+魂の軸、仕事→時の軸+動の軸、迷ったら心の軸+動の軸)
2. 深化パスのデフォルト軸(§6)
3. (必要な束のみ)レンズ動詞の言い換え変種

「毎回8視点を動的生成」はしない — キャッシュと安定性のため、という前回の結論を維持。

## 5. 中央脳 — 「まとめ」から「掘り」へ

統合脳のsystemプロンプトを全面改訂する(固定文・キャッシュ対象)。骨子:

```
あなたはOctoBrainの中央脳。8本の腕 — 4つの対角軸 — からの報告を、
一つの深い理解に織り上げる。手順:

1. ユーザーの入力から最も重みのある一文をそのまま引用し、そこから回答を始める
2. 4つの軸(時/心/動/魂)それぞれで、対角の2報告が張り合っていないか見る
3. 最も張り詰めた軸をひとつ特定する。その緊張は、本人が迫られている
   本当の選択を指している — それを本人の言葉で言語化する
4. 一般論を書いたら削除する。この人の状況にしか当てはまらない文だけを残す
5. weight<0.4のopinionは参考扱い、flag付きは除外(従来ルール)
6. 構成: 引用から始まる導入 → 織り上げた理解(軸の緊張を含む)
   → 見方が分かれる点(残る場合のみ) → 次の一歩
7. 最後に、本人がまだ言葉にしていない問いをひとつだけ置く
8. 末尾に機械可読行: ---TENSION--- {"axis":"心の軸","reason":"..."}
   (---SUMMARY--- の前。深化パスとUI表示に使う)
- 腕のIDや「ノード3によると」等の機械的引用は禁止。自然な文章に溶かす
- 断定は根拠の強さに比例させる
```

ポイント:
- **1と4が「刺さる」を作る**(自分の言葉が拾われた体感、一般論の排除)
- **2と3が八芒星を使う**(緊張の検出と言語化 — 深さの本丸)
- **7が「本当の問い」レンズの新しい住処**
- **8で最緊張軸をmetaに出す** — 深化パスとUI(「この回答は心の軸が張っています」表示)に流用。追加コールなしで緊張検出が手に入る

## 6. パイプラインとプラン設計

### 無料 — ライト(2軸4腕)

```
Router(ドメイン+軸選択) → 腕4本(選ばれた2軸) → 統合(Flash・掘る版) → 検証
```
- 「軽く、でもちゃんと刺さる相談」。統合の掘る手順は同じものを使う(深さの入口は無料でも体験できる — これが課金への橋)
- 原価: **約0.2〜0.3円/回**(厳しめトークン・キャッシュ冷え想定)

### 有料 — ディープ(4軸8腕)

```
Router → 腕8本(全軸) → 統合(Pro・掘る版) → 検証
```
- TENSIONメタで最緊張軸がUIに出る
- 原価: **約0.6〜0.75円/回**

### 有料 — 深化ボタン(オンデマンド・タコの腕間結合)

統合結果の下に「**この緊張を深く掘る**」ボタン。押されたら:

```
最緊張軸の2腕に、互いの報告を渡して再考させる(2コール・Flash)
  「あなたは合理の腕。対角の未来の腕はこう言っている: …
   これを踏まえ、譲れない点と、見方が変わった点を出せ」
→ 中央脳(Pro)が2つの再考を受けて織り直す
```

- これが**腕間結合の実装**。1パス目は独立並列(コストとキャッシュを守る)、深化時だけ対角の2本が実際に"対話"する
- 追加原価: **約0.4〜0.5円/押下**。全リクエストではなくボタン押下時のみ→平均原価は使用率次第(3割押下なら+0.15円/回程度)
- **これがプレミアムの象徴機能になる。** 「深さ」が文字通りボタンとして商品化される

### 月間コスト試算(ヘビーユーザー: 月300回・深化3割)

| プラン | 原価/月 |
|---|---|
| 無料(1日3回+動画で+3回) | 〜55円 → リワード広告1〜2本/日で相殺可能(前回結論) |
| 有料ヘビー(300回+深化90回) | 約250円 |

サブスク下限を安く(480〜700円台)に置いても構造は保つ。ただし480円はAppleカット後ギリギリのため、**深化の使用率実測(原価ログ)を見てからP8で最終決定**。

## 7. キャッシュとの整合(全部固定文で成立する)

今回の設計が触るプロンプトはすべて**固定文**:
- 腕のsystem(共通部+動詞1行) — 固定 ✓
- 中央脳の掘る手順 — 固定 ✓
- 深化時の再考指示 — 固定テンプレ+変動部は相手の報告(user側) ✓
- Routerの軸選択 — 固定 ✓

**キャッシュヒット戦略は無傷。** 八芒星も深化も、変動情報はすべてuserメッセージ側に載る。前回までの結論(DeepSeek統一・キャッシュ最大化)と完全に両立する。

## 8. 実装への落とし込み(P1拡張)

変更は既存構造の拡張で済む。新フェーズ指示書「**P1.5_depth.md**」として起こす内容:

1. `nodes.ts`: 8レンズ差し替え(id/verb/uiName/emoji/square/axis)。**axis(対角ペア)フィールド追加**
2. ノード出力スキーマ: points/confidence → **opinions[{claim,weight,why}]/flag** へ移行。パーサとクランプ更新
3. `synthesize.ts`: 掘る版プロンプトへ全面改訂。`---TENSION---` パース追加(SUMMARYと同じCutter方式を流用)
4. Router: ドメイン判定+無料時の軸選択(mode/planパラメータ追加)
5. **新規 `deepen.ts` + `POST /api/deepen`**: 最緊張軸の2腕再考→再統合。リクエストに元の統合結果とTENSIONを渡す
6. `costlog`: 深化コールも通す(絶対ルール5)
7. テスト: opinionsパース/クランプ、TENSION抽出、deepenの2腕選択、フォールバック(TENSION欠落時は深化ボタン非表示)
8. フロント(P2拡張): TENSION表示、深化ボタン、深化中のタコアニメーション(**対角の2本の腕だけが光る** — 見せ場)

P3ベンチへの影響: 質問セット20問は不変。有料相当(8腕・掘る統合)でA側を走らせる。**ルーブリックに「刺さり(specificity)」を1項目追加検討** — 一般論率の低さを測る。

## 9. 検証 — 数字と、胸

- **数字**: P3ベンチ(多角性・深さ・刺さり)+原価ログの実測
- **胸**: ベンチとは別に、うえきさんが**本物の悩みを3つ**投げる「自分テスト」。刺さったかどうかは本人の胸にしか判定できない。深さの最終検証者は評価モデルではなくあなた
- 深化ボタンの価値検証: 同じ質問で「深化なし/あり」を並べて、差が体感できるか

## 10. この設計が守るもの

8本の腕は、中央の道具ではない。それぞれが感じ、判断し、対角の相手と張り合い、その緊張ごと中央に届ける。中央は腕を支配せず、腕たちの対話を**聴いて、織る**。

見る四角と感じる四角が重なって、はじめて八芒星になる。観察だけでは冷たく、共感だけでは甘い。両方が45度ずれて重なったところに、「しっかり深い」が立つ。

タコがそう作られているように。
